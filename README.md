# 🧰 AI Playbook for Cursor (Command Collection)

> **Modular | Independent | Surgical**
>
> 这是一套为 Cursor 打造的**独立指令集和rules规则**。
> 即插即用的“功能增强插件”。你可以根据当前任务需求，随时调用特定指令来辅助开发。


## 🛠 独立指令模块 (Independent Modules)

你可以像使用命令行工具一样，在 Cursor Chat 中单独使用以下指令：

### 1. 代码重构工具
当你觉得代码写得乱，或者需要优化结构时使用。

* **指令**：`@refactor` (或输入 `/refactor`)
* **功能**：**交互式代码清洗**。
    * AI 不会直接改代码，而是先生成一张**诊断表**（列出属性爆炸、逻辑嵌套、冗余定义等问题）。
    * **适用场景**：接手“屎山”代码、优化复杂组件逻辑。

### 2. 接口对接工具
当你拿着后端的 API 响应数据，需要渲染到前端页面时使用。

* **指令**：`@api-bind` (或输入 `/api-bind`)
* **功能**：**智能字段映射**。
    * 自动处理后端 `snake_case` 到前端 `camelCase` 的转换。
    * 自动判断数据复杂度：简单数据用解构，复杂嵌套数据直接映射，防止写出一堆无用的转换代码。
    * **适用场景**：联调接口、填充页面数据。

### 3. 类型清理工具
当 TypeScript 类型定义混乱、重复或满屏 `any` 时使用。

* **指令**：`@refactor-types` (或输入 `/refactor-types`)
* **功能**：**类型归位**。
    * 智能识别类型应该放哪里（是复用现有的 `types/User.ts` 还是新建）。
    * 解决类型命名冲突和内联定义问题。
    * **适用场景**：TS 项目重构、清理类型报错。

### 4. Zod Schema 生成器
当你需要对后端数据做严格校验时使用。

* **指令**：`@zod-gen` (或输入 `/zod-gen`)
* **功能**：**生成校验层**。
    * 把一段 JSON 数据扔给它，它会自动生成严格的 `zod` schema 定义。
    * 包含自动的字段重命名（Transform）和类型推导。
    * **适用场景**：表单验证、API 数据清洗。

## 🧠 记忆增强模块 (Memory Module)

如果你希望 AI 能够“记住”你的项目背景和进度，可以使用这组独立的记忆管理命令。

* **初始化**：`/init-momory <tag>`
    * **作用**：在 `.cursor/memory-<tag>/` 下创建记忆文件结构（项目简介、进度表）。
* **同步记忆**：`/memo`
    * **作用**：把当前的对话结论同步写入到记忆文件中，防止遗忘。

## 🤖 模型选型参考 (2026 Edition)

建议根据不同的指令选择不同的模型：

| 任务类型 | 推荐模型 | 原因 |
| :--- | :--- | :--- |
| **日常编码 / 快速原型** | **Composer 1** | 速度最快，肌肉记忆好。 |
| **架构设计 / 复杂重构** | **Claude 4.5 Opus** | 逻辑推理最强，适合处理 `/refactor` 的诊断。 |
| **前端样式 / UI 还原** | **Claude 4.5 Sonnet** | 审美最好，懂 CSS 细节。 |
| **算法 / 高安全逻辑** | **GPT-5.2** | 数理逻辑严密，适合 `/zod-gen` 或复杂算法。 |
